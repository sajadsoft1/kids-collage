# چک‌لیست تست دستی صفحه Certificate Builder

## آدرس‌ها
- **لیست قالب‌ها:** `/admin/certificate-template`
- **ایجاد قالب:** `/admin/certificate-template/create`
- **ویرایش قالب:** `/admin/certificate-template/{id}/edit`

## پیش‌نیاز
- ورود به پنل ادمین با کاربری که دسترسی ماژول `education.certificates` دارد.

---

## ۱. تب «اطلاعات پایه» (Basic)
- [ ] عنوان (Title) اجباری است و بدون آن ذخیره نشود.
- [ ] نامک (Slug) اختیاری است؛ در صورت خالی بودن از عنوان ساخته شود.
- [ ] نام آموزشگاه (Institute name) اختیاری است.
- [ ] انتخاب چیدمان (Layout): Classic / Minimal / Custom.
- [ ] سوئیچ «قالب پیش‌فرض» (Default template) درست کار کند.
- [ ] دکمه ذخیره بدون خطا کار کند و به لیست برگردد.

## ۲. تب «طراحی» (Design)
- [ ] متن هدر (Header text) با placeholderها (مثلاً `{{student_name}}`) ذخیره شود.
- [ ] متن بدنه (Body text) و متن فوتر (Footer text) ذخیره شوند.
- [ ] لیست placeholderها در سایدبار نمایش داده شود.
- [ ] راهنما (hint) برای placeholderها نمایش داده شود.

## ۳. تب «رسانه» (Media)
- [ ] آپلود لوگو (Logo) با فرمت‌های مجاز (jpeg, png, gif, webp).
- [ ] آپلود تصویر پس‌زمینه (Background).
- [ ] آپلود تصویر امضا (Signature).
- [ ] پس از آپلود، پیش‌نمایش تصاویر نمایش داده شود.
- [ ] در حالت ویرایش، تصاویر قبلی نمایش داده شوند.

## ۴. تب «پیش‌نمایش» (Preview)
- [ ] با پر کردن متن‌ها در تب طراحی، پیش‌نمایش با داده نمونه (مثلاً John Doe، Sample Course) به‌درستی نمایش داده شود.
- [ ] placeholderها در پیش‌نمایش با مقادیر نمونه جایگزین شوند.

## ۵. لیست قالب‌ها (Index)
- [ ] از منوی ادمین (Certificate Builder) به لیست بروید.
- [ ] دکمه «ایجاد قالب گواهینامه» به صفحه create برود.
- [ ] دکمه ویرایش هر ردیف به صفحه edit برود.
- [ ] دکمه حذف با تأیید کار کند (قالب پیش‌فرض نباید حذف شود).
- [ ] فیلترها (پیش‌فرض، چیدمان، تاریخ) کار کنند.

## ۶. یکپارچگی با دوره (Course Template)
- [ ] در صفحه ویرایش/ایجاد دوره (Course Template)، فیلد «قالب‌های گواهینامه» نمایش داده شود.
- [ ] انتخاب یک قالب گواهینامه برای دوره ذخیره شود.

## ۷. رندر و UI
- [ ] breadcrumb و دکمه بازگشت درست کار کنند.
- [ ] تغییر تب بدون خطا انجام شود.
- [ ] فرم‌ها با قوانین mary-ui و بدون خطای console در مرورگر نمایش داده شوند.
